## 🧠 核心概念

### 什么是交叉编译？

```
┌─────────────────────────────────────────────────────┐
│                    x86 PC（宿主机）                    │
│                                                     │
│   hello.c ──→ arm-gcc ──→ hello_dry_driver.ko       │
│                 ▲                    │               │
│                 │                    │  NFS 共享      │
│          交叉编译器              ┌───▼──────────┐     │
│                                 │  ARM 开发板    │     │
│                                 │               │     │
│                                 │ insmod → 运行  │     │
│                                 └───────────────┘     │
└─────────────────────────────────────────────────────┘
```

**交叉编译**：在一种架构（x86）的机器上，编译出另一种架构（ARM）的可执行程序。

---

## 🔧 交叉编译三要素

### 1. 目标板内核源码

```makefile
LINUX_KERNEL_PATH = /home/king/code/kernel/linux-4.1.15-elf1
```

- 必须是和目标开发板**内核版本一致**的源码
- 用于获取内核头文件、模块构建脚本

### 2. 交叉编译器

```makefile
CROSS_COMPILE = arm-poky-linux-gnueabi-
```

常见交叉编译器前缀对照：

| 编译器前缀 | 来源 | 适用场景 |
|-----------|------|---------|
| `arm-linux-gnueabihf-` | Linaro/Ubuntu | 通用 ARM 硬浮点 |
| `arm-linux-gnueabi-` | 通用 | 通用 ARM 软浮点 |
| `arm-poky-linux-gnueabi-` | Yocto Project | i.MX 系列 |
| `aarch64-linux-gnu-` | 通用 | ARM 64位 |
| `arm-none-eabi-` | ARM 官方 | 裸机开发 |

### 3. 目标板 sysroot

```makefile
CC = $(CROSS_COMPILE)gcc --sysroot=/home/king/code/rootfs/rootfs
```

sysroot 是目标板根文件系统在本地的副本，编译器从这里获取头文件和库。

## 📡 NFS 共享目录配置

NFS（Network File System）是 Linux 下最常用的网络文件共享方案。

### 宿主机端（Ubuntu）

```bash
# 安装 NFS 服务
sudo apt install nfs-kernel-server

# 编辑共享配置
sudo vi /etc/exports
# 添加你的工作目录（替换为你自己的路径）：
/home/king/project *(rw,sync,no_subtree_check,no_root_squash)

# 重启 NFS 服务
sudo systemctl restart nfs-kernel-server
```

### 开发板端

```bash
# 创建挂载目录
mkdir -p /mnt/king

# 手动挂载
mount -t nfs -o nolock,vers=3 192.168.100.101:/home/king/project /mnt/king

# 验证挂载
ls /mnt/king

# 取消挂载（如需）
umount /mnt/king
```

### 永久挂载（开发板重启后依然生效）

```bash
vi /etc/fstab
# 添加一行：
# 192.168.100.101:/home/king/project /mnt/king nfs nolock,vers=3 0 0
```

## 🔨 完整工作流

```
┌──────────────────────────────────────────────────────────────┐
│                     Step-by-Step 流程                         │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. PC 上编写代码                                             │
│     hello.c → 编辑你的驱动代码                                 │
│                          │                                    │
│                          ▼                                    │
│  2. PC 上交叉编译                                             │
│     make → 生成 hello_dry_driver.ko                          │
│                          │                                    │
│                          ▼                                    │
│  3. 文件传输（NFS 方式）                                       │
│     .ko 文件已在共享目录，开发板可直接访问                       │
│                          │                                    │
│                          ▼                                    │
│  4. 开发板上加载测试                                           │
│     insmod hello_dry_driver.ko → dmesg 查看输出              │
│                          │                                    │
│                          ▼                                    │
│  5. 开发板上卸载                                              │
│     rmmod hello_dry_driver → dmesg 确认退出                   │
│                          │                                    │
│                          ▼                                    │
│  6. 修改代码，重新编译，重复...                                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 📋 常用命令速查

| 命令 | 说明 | 在何处执行 |
|------|------|-----------|
| `make` | 交叉编译模块 | 宿主机 |
| `make clean` | 清理编译产物 | 宿主机 |
| `insmod xxx.ko` | 加载模块 | 开发板 |
| `rmmod xxx` | 卸载模块（无 .ko） | 开发板 |
| `lsmod` | 列出已加载模块 | 开发板 |
| `dmesg \| tail` | 查看最后日志 | 开发板 |
| `dmesg -w` | 实时跟踪日志 | 开发板 |
| `dmesg -wT` | 带时间戳实时跟踪 | 开发板 |
| `mount \| grep nfs` | 查看 NFS 挂载 | 开发板 |
| `mount -t nfs -o nolock,vers=3 IP:/path /mnt/xxx` | 手动挂载 NFS | 开发板 |
| `umount /mnt/xxx` | 取消挂载 | 开发板 |
| `umount -l /mnt/xxx` | 强制取消挂载 | 开发板 |

## 🐛 常见问题

### Q1: make 报错 "No rule to make target"

**原因**：Makefile 中 `obj-m` 的名字和 C 文件名不匹配。

```makefile
obj-m := hello_dry_driver.o   # 对应 hello_dry_driver.c
```

### Q2: insmod 报错 "Invalid module format"

**原因**：编译器的架构参数与开发板 CPU 不匹配。

```bash
# 检查开发板 CPU 信息
cat /proc/cpuinfo

# 检查 .ko 文件架构
file hello_dry_driver.ko
```

### Q3: insmod 报错 "version magic mismatch"

**原因**：内核源码版本与开发板运行的内核版本不一致。

```bash
# 开发板上查看内核版本
uname -r

# 确保 LINUX_KERNEL_PATH 指向相同版本
```

### Q4: NFS 挂载失败

```bash
# 检查网络是否通
ping 192.168.100.101

# 检查宿主机 NFS 服务状态
sudo systemctl status nfs-kernel-server

# 开发板尝试不同参数挂载
mount -t nfs -o nolock,vers=3 192.168.100.101:/home/king/project /mnt/king
```
