# 00 - 开发环境搭建

本教程介绍如何搭建 Linux 驱动开发环境。

## 📋 环境要求

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| 操作系统 | Ubuntu 18.04 / CentOS 7 | Ubuntu 22.04 LTS |
| 内存 | 4 GB | 8 GB+ |
| 磁盘空间 | 20 GB | 50 GB+ |
| CPU | 双核 | 四核+ |

> ⚠️ **强烈建议使用虚拟机进行驱动开发**，避免对宿主系统造成不可逆的影响。

---

## 🪟 Windows 用户

### 方案一：WSL2（推荐）

Windows 10/11 用户可以使用 WSL2 作为开发环境。

```powershell
# 以管理员身份打开 PowerShell，安装 WSL2
wsl --install

# 重启后，设置 WSL 用户名和密码
# 安装编译工具链
sudo apt update
sudo apt install build-essential linux-headers-$(uname -r)
```

### 方案二：虚拟机

使用 VirtualBox 或 VMware 虚拟机安装 Ubuntu。

---

## 🐧 Ubuntu / Debian

```bash
# 更新软件源
sudo apt update

# 安装基础编译工具
sudo apt install build-essential

# 安装内核头文件（关键！）
sudo apt install linux-headers-$(uname -r)

# 验证安装
ls /lib/modules/$(uname -r)/build
```

---

## 🎩 CentOS / RHEL / Fedora

```bash
# 安装编译工具
sudo yum groupinstall "Development Tools"

# 安装内核头文件
sudo yum install kernel-devel-$(uname -r)

# 验证安装
ls /lib/modules/$(uname -r)/build
```

---

## ✅ 验证环境

执行以下命令验证开发环境是否就绪：

```bash
# 1. 检查 GCC 版本
gcc --version
# 预期输出：gcc version x.x.x

# 2. 检查 Make 版本
make --version
# 预期输出：GNU Make x.x.x

# 3. 检查内核版本
uname -r
# 预期输出：5.x.x-xx-generic 或类似

# 4. 检查内核头文件
ls /lib/modules/$(uname -r)/build
# 预期输出：包含 Kconfig, Makefile, Module.symvers 等文件
```

## 🔧 常用命令速查

| 命令 | 说明 |
|------|------|
| `insmod xxx.ko` | 加载内核模块 |
| `rmmod xxx` | 卸载内核模块（不带 .ko） |
| `lsmod` | 查看已加载的模块列表 |
| `modinfo xxx.ko` | 查看模块信息 |
| `dmesg` | 查看内核日志 |
| `dmesg -w` | 实时查看内核日志（Ctrl+C 退出） |

## 🐛 常见问题

### Q1: 找不到内核头文件

```bash
# 错误信息：/lib/modules/xxx/build: No such file or directory

# 解决：安装对应版本的内核头文件
sudo apt install linux-headers-$(uname -r)
```

### Q2: 编译失败，版本不匹配

确保内核版本与头文件版本一致：

```bash
# 查看当前运行的内核版本
uname -r

# 查看已安装的头文件版本
ls /lib/modules/

# 如果版本不一致，需要安装匹配的头文件
sudo apt install linux-headers-$(uname -r)
```

### Q3: 权限不足

加载/卸载内核模块需要 root 权限：

```bash
# 使用 sudo
sudo insmod hello.ko
sudo rmmod hello

# 或者切换到 root 用户
sudo su
insmod hello.ko
```
